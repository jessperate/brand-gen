Installation steps
01
Check your macOS version

First, let's open Terminal. It's a plain text window where you type commands — think of it like texting your computer. Press ⌘ + Space (that's the Command key and the spacebar at the same time), type Terminal, and press Enter. A black or white window will open. That's it — you're in.

Don't worry if it looks intimidating. You'll mostly just be copying and pasting commands from this guide.

To make sure your Mac is new enough: click the Apple menu (top-left corner of your screen) → About This Mac. If your Mac is from 2016 or later, you're good to go.

02
Install Homebrew

Homebrew is basically an app store for developer tools that lives in your terminal. Instead of downloading and installing things manually, you just type one command and Homebrew handles everything. You'll use it in the next few steps.

Not sure if you already have it? Type brew --version in Terminal and press Enter. If you see a version number, skip to Step 3. If you see "command not found," run the command below.

Terminal — zsh
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
==> Checking for sudo access (which may request your password)...
==> This script will install:
   /usr/local/bin/brew
==> Installation successful!
💡
When it asks for your password, type it and press Enter. Nothing will show up on screen as you type — no dots, no asterisks. That's completely normal, it's a security feature. Just type your Mac login password and press Enter. The install takes 3–5 minutes.
03
Install Node.js

Node.js is the engine that lets Claude Code run on your Mac. You don't need to understand what it does — just think of it as a required piece of software, like needing Java to run certain apps. Run the two commands below to install it and confirm it worked.

Terminal — zsh
$ brew install node
$ node --version
v22.14.0 # any v18+ is fine
$ npm --version
10.9.2
✅
Both commands should print a version number (something like v22.14.0). If you see "command not found" instead, close the Terminal window completely, reopen it, and try the commands again. That usually fixes it.
04
Install Claude Code

This is the main event. The command below downloads and installs Claude Code onto your Mac. It's one line — copy it exactly, paste it into Terminal, and press Enter. It takes about a minute.

Terminal — zsh
$ curl -fsSL https://claude.ai/install.sh | bash
==> Downloading Claude Code...
==> Installing to /usr/local/bin/claude
==> Claude Code installed successfully
$ claude --version
2.1.71
⚠️
Use the command above, not anything starting with sudo npm install. If you've seen that pattern online, ignore it here — it causes permission errors that are annoying to undo. Stick with the curl command above.
05
Authenticate with your Claude account

Type claude and press Enter. Claude Code will pop open a browser window and ask you to log in — use the same email and password you use on claude.ai. Once you authorize it in the browser, flip back to Terminal and you should see a success message.

After logging in, run claude doctor — it's a built-in health check that confirms everything is working.

Terminal — zsh
$ claude
Opening browser for authentication...
https://claude.ai/auth/cli?token=xxxx
# hold ⌘ + click to open if it doesn't launch automatically
✓ Authenticated successfully
✓ Connected to Claude claude-sonnet-4-6
$ claude doctor
✓ Node.js v22.14.0
✓ Authentication valid
✓ Network connection ok
✓ All checks passed
💡
All four checkmarks = you're good to go. claude doctor is your best friend when something feels off later — run it first before anything else, and it'll usually tell you exactly what's wrong.
06
Install Git and connect GitHub

Git is like "track changes" in Google Docs, but for code. Every time you save a meaningful update, Git records it so you can always go back. GitHub is the online service where those records live.

The commands below check if Git is already installed, set it up with your name and email, and then create a secure key that lets your Mac talk to GitHub without a password every time. Follow them in order.

Terminal — zsh
$ git --version
git version 2.47.1 # if you see this, skip brew install below
$ brew install git # only if not installed
$ git config --global user.name "Your Name"
$ git config --global user.email "[email protected]"
$ ssh-keygen -t ed25519 -C "[email protected]"
Generating public/private ed25519 key pair.
Enter file to save the key: # press Enter to accept default
$ cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... [email protected]
# copy this entire line, then add it to github.com/settings/keys
🔑
One more thing: the last command prints a long string of text starting with ssh-ed25519. Copy that entire line. Then go to github.com → your profile photo (top right) → Settings → SSH and GPG Keys → New SSH Key, give it any name (like "My MacBook"), paste in the text, and save. This is what lets GitHub trust your computer. You only do this once.
07
Install Vercel CLI and connect your account

Right now Vercel lives in your browser. This step installs a "shortcut" version of Vercel that lives in your terminal. Once it's set up, you can deploy any project to a live URL by typing one word: vercel.

Log in with the same GitHub account you just set up — they'll connect automatically.

Terminal — zsh
$ npm install -g vercel
added 1 package in 3s
$ vercel login
? Log in to Vercel Continue with GitHub
Opening browser... # authorize in the browser, then return here
✓ Logged in as [email protected]
$ vercel --version
Vercel CLI 41.x.x
08
Start using Claude Code

You're set up. Here's how to actually use it.

Navigate to a project folder in Terminal (use cd folder-name to move around, or drag a folder from Finder into Terminal to paste its path automatically). Then type claude and press Enter.

Claude reads all your files and waits for you to tell it what to do — in plain English. Before it changes anything, it shows you exactly what it's about to do and asks for your approval. You're always in control.

Terminal — zsh — your-project
$ cd ~/projects/your-project
$ claude
✻ Welcome to Claude Code!
Type your request below. Ctrl+C to exit.
> Explain what this project does
Reading codebase...
This is a Next.js application that... (Claude responds here)
🚀
Good first prompts to try: "Explain what this project does" / "What files are in here?" / "Help me add a new page" / "Something's broken — can you find it?" — You can talk to it like a person. It's not picky about phrasing.
Terminal quick reference
Command What it does
cd folder-name Navigate into a folder
cd .. Go up one level
pwd Show where you are right now
ls List files in current folder
claude doctor Run a health check on your Claude Code setup
claude --version Check which version is installed
vercel Deploy current project to Vercel
⌘ + K Clear terminal window
Ctrl + C Stop whatever is running / exit Claude
Troubleshooting

Something not working? Don't panic — 90% of issues come from one of the things below. Try each fix in order and you'll almost certainly unblock yourself. If not, drop a message in #tools-and-infra and paste whatever error you're seeing.

01
"command not found: claude" after installation

This is the most common issue people hit and it's an easy fix. Claude installed fine — your terminal just doesn't know about it yet because it only checks for new software when it starts. You need to either restart Terminal or run one command to refresh it.

Fix 1 — Restart Terminal (simplest): Quit Terminal completely (⌘ + Q), reopen it, and try claude --version again.

Fix 2 — Or run this to refresh without restarting:

Terminal — zsh
$ source ~/.zshrc # reload shell config
$ claude --version
2.1.71 # if this works, you're done

Fix 2 -- If that doesn't work, manually add Claude to your PATH:

Terminal — zsh
$ echo $PATH | tr ':' '\n' | grep -E "(local|claude)"
# you should see /usr/local/bin or ~/.local/bin listed
# if not, run this:
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
$ claude --version
💡
If you're on bash instead of zsh, replace ~/.zshrc with ~/.bashrc in all commands above. Check your shell with echo $SHELL.
02
EACCES permission denied error

"Permission denied" means your Mac is blocking the install from writing files to a protected folder. It's a safety feature, but it gets in the way here. The commands below create a new folder for these tools and tell your Mac to use that one instead — it doesn't delete anything, just redirects.

Terminal — zsh
$ mkdir ~/.npm-global
$ npm config set prefix '~/.npm-global'
$ echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
$ source ~/.zshrc
$ curl -fsSL https://claude.ai/install.sh | bash # reinstall
03
Node.js version too old (or not found)

Claude Code requires a recent version of Node.js (version 18 or higher). If you've had Node installed for a while, it might be out of date. Run node --version to check — the number after the "v" needs to be 18 or above. If it's lower, run the upgrade command below.

Terminal — zsh
$ node --version
v16.14.0 # too old -- need v18+
$ brew upgrade node # if installed via Homebrew
$ node --version
v22.14.0 ✓
💡
If you manage multiple Node versions (common for devs), use nvm use 20 or nvm install --lts to switch to a supported version.
04
Authentication fails or browser doesn't open

A few things can cause this. Check them in order — most people get unblocked at step 1 or 2.

Check 1: Make sure you're on a paid Claude plan. Go to claude.ai and check your account settings. Free accounts can't use Claude Code — this is the most common cause of auth failures.

Check 2: If a browser window didn't pop up automatically, look at the Terminal output. There should be a URL starting with https://claude.ai/auth/... — hold ⌘ and click it, or copy/paste it into your browser manually.

Terminal — zsh
$ claude
Opening browser for authentication...
https://claude.ai/auth/cli?token=xxxx
# if browser doesn't open: copy the URL and paste it manually
# or: hold ⌘ and click the link in Terminal
$ claude doctor # run this after auth to verify
✓ Authentication valid
⚠️
Still failing? Check the Anthropic status page -- an API incident accounts for roughly 5% of authentication failures that look like local problems.
05
Xcode / command line tools error on Mac

macOS needs a set of built-in developer tools (called Xcode Command Line Tools) before it can install almost anything else. You might not have these if your Mac is new or freshly wiped. If you see any error mentioning xcode-select, run the command below, wait for a popup dialog to appear, click Install, and let it finish before trying anything else. It takes about 5 minutes.

Terminal — zsh
$ xcode-select --install
xcode-select: note: install requested for command line developer tools
# a dialog will pop up -- click Install and wait ~5 minutes
$ xcode-select -p # verify it worked
/Library/Developer/CommandLineTools ✓
06
Claude installs but nothing happens when you run it

If typing claude does nothing, blinks and exits, or hangs without showing anything, the install probably got interrupted partway through. The cleanest fix is to uninstall and reinstall from scratch. Your settings and login are saved in a separate folder (~/.claude/) and won't be touched.

Terminal — zsh
$ which claude # find the install location
/usr/local/bin/claude
$ npm uninstall -g @anthropic-ai/claude-code
$ curl -fsSL https://claude.ai/install.sh | bash
$ claude doctor
✓ All checks passed
💡
Your config in ~/.claude/ is preserved during reinstall, so you won't lose any settings or authentication.
07
Still stuck? Run the full diagnostic

If none of the above fixed it, run the commands below. They check all the important things on your system and print a report. Copy everything that appears in Terminal and paste it into #tools-and-infra on Slack — someone will help you sort it out.

Terminal — zsh
$ echo "=== Shell ===" && echo $SHELL
$ echo "=== Node ===" && node --version
$ echo "=== npm ===" && npm --version
$ echo "=== Claude path ===" && which claude
$ echo "=== Claude version ===" && claude --version
$ claude doctor
# copy everything above and paste it in #tools-and-infra
