Installation steps
01
Check your macOS version

Claude Code requires macOS 10.15 or later. If your Mac is from the last 5 years, you're fine. To verify: Apple menu → About This Mac.

Then open Terminal. Press ⌘ + Space, type "Terminal", hit Enter.

02
Install Homebrew

Homebrew is a package manager for Mac. It makes installing developer tools much easier. Skip this step if you already have it (brew --version will tell you).

Terminal — zsh
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
==> Checking for sudo access (which may request your password)...
==> This script will install:
   /usr/local/bin/brew
==> Installation successful!
💡
When prompted for your password, type it and press Enter. Nothing will appear on screen -- that's normal. Homebrew will take a few minutes.
03
Install Node.js

Node.js lets you run JavaScript-based tools like Claude Code on your machine. You need version 18 or higher.

Terminal — zsh
$ brew install node
$ node --version
v22.14.0 # any v18+ is fine
$ npm --version
10.9.2
✅
Both commands should return a version number. If you see "command not found," close Terminal, reopen it, and try again.
04
Install Claude Code

The native installer is the recommended path. One command, no extra dependencies, auto-updates in the background.

Terminal — zsh
$ curl -fsSL https://claude.ai/install.sh | bash
==> Downloading Claude Code...
==> Installing to /usr/local/bin/claude
==> Claude Code installed successfully
$ claude --version
2.1.71
⚠️
Don't use sudo npm install -g. This causes permission issues and security risks. Use the curl installer above, or Homebrew: brew install --cask claude-code
05
Authenticate with your Claude account

Run Claude for the first time. It will open a browser window for you to log in with your Claude.ai credentials.

Terminal — zsh
$ claude
Opening browser for authentication...
https://claude.ai/auth/cli?token=xxxx
# hold ⌘ + click to open if it doesn't launch automatically
✓ Authenticated successfully
✓ Connected to Claude claude-sonnet-4-6
$ claude doctor
✓ Node.js v22.14.0
✓ Authentication valid
✓ Network connection ok
✓ All checks passed
💡
claude doctor runs a health check on your setup. Run it anytime something feels broken -- it auto-detects most issues and tells you how to fix them.
06
Install Git and connect GitHub

Git tracks your code changes. GitHub is where those changes live online. Check if Git is already installed first, then configure it with your details.

Terminal — zsh
$ git --version
git version 2.47.1 # if you see this, skip brew install below
$ brew install git # only if not installed
$ git config --global user.name "Your Name"
$ git config --global user.email "[email protected]"
$ ssh-keygen -t ed25519 -C "[email protected]"
Generating public/private ed25519 key pair.
Enter file to save the key: # press Enter to accept default
$ cat ~/.ssh/id_ed25519.pub
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... [email protected]
# copy this entire line, then add it to github.com/settings/keys
🔑
Adding your SSH key to GitHub: Go to github.com → Settings → SSH and GPG Keys → New SSH Key. Paste the output from the last command above. Full walkthrough at docs.github.com/en/authentication/connecting-to-github-with-ssh
07
Install Vercel CLI and connect your account

Vercel is how you deploy projects from the terminal. Install the CLI, log in once, and after that vercel from any project folder kicks off a deploy.

Terminal — zsh
$ npm install -g vercel
added 1 package in 3s
$ vercel login
? Log in to Vercel Continue with GitHub
Opening browser... # authorize in the browser, then return here
✓ Logged in as [email protected]
$ vercel --version
Vercel CLI 41.x.x
08
Start using Claude Code

Navigate to any project folder and run claude. It reads your files and waits for instructions. Every proposed change shows as a diff -- you approve or reject before anything happens.

Terminal — zsh — your-project
$ cd ~/projects/your-project
$ claude
✻ Welcome to Claude Code!
Type your request below. Ctrl+C to exit.
> Explain what this project does
Reading codebase...
This is a Next.js application that... (Claude responds here)
🚀
Good first prompts: "Explain what this project does" / "Write tests for app.js" / "Fix the error in this file" / "Help me set up a new Next.js project"
Terminal quick reference
Command What it does
cd folder-name Navigate into a folder
cd .. Go up one level
pwd Show where you are right now
ls List files in current folder
claude doctor Run a health check on your Claude Code setup
claude --version Check which version is installed
vercel Deploy current project to Vercel
⌘ + K Clear terminal window
Ctrl + C Stop whatever is running / exit Claude
Troubleshooting

These are the most common roadblocks people hit. Work through them in order -- most issues are resolved by step 2 or 3.

01
"command not found: claude" after installation

This is the most common issue. The installer added Claude to your PATH, but your current terminal session hasn't picked it up yet.

Fix 1 -- Reload your shell config (try this first):

Terminal — zsh
$ source ~/.zshrc # reload shell config
$ claude --version
2.1.71 # if this works, you're done

Fix 2 -- If that doesn't work, manually add Claude to your PATH:

Terminal — zsh
$ echo $PATH | tr ':' '\n' | grep -E "(local|claude)"
# you should see /usr/local/bin or ~/.local/bin listed
# if not, run this:
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
$ source ~/.zshrc
$ claude --version
💡
If you're on bash instead of zsh, replace ~/.zshrc with ~/.bashrc in all commands above. Check your shell with echo $SHELL.
02
EACCES permission denied error

This happens when npm tries to write to a system directory it doesn't have access to. Usually caused by using sudo npm install at some point, which locks down folder permissions. The fix is to redirect npm's global directory to your user folder.

Terminal — zsh
$ mkdir ~/.npm-global
$ npm config set prefix '~/.npm-global'
$ echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
$ source ~/.zshrc
$ curl -fsSL https://claude.ai/install.sh | bash # reinstall
03
Node.js version too old (or not found)

Claude Code needs Node 18 or higher. If you installed Node a long time ago, you may be on an older version. Check first, then update if needed.

Terminal — zsh
$ node --version
v16.14.0 # too old -- need v18+
$ brew upgrade node # if installed via Homebrew
$ node --version
v22.14.0 ✓
💡
If you manage multiple Node versions (common for devs), use nvm use 20 or nvm install --lts to switch to a supported version.
04
Authentication fails or browser doesn't open

A few things can cause this. Work through them in order.

Check 1: Make sure you're on a paid Claude plan. Free accounts can't use Claude Code.

Check 2: If the browser didn't open automatically, copy the URL manually.

Terminal — zsh
$ claude
Opening browser for authentication...
https://claude.ai/auth/cli?token=xxxx
# if browser doesn't open: copy the URL and paste it manually
# or: hold ⌘ and click the link in Terminal
$ claude doctor # run this after auth to verify
✓ Authentication valid
⚠️
Still failing? Check the Anthropic status page -- an API incident accounts for roughly 5% of authentication failures that look like local problems.
05
Xcode / command line tools error on Mac

macOS requires Xcode Command Line Tools for many installs, including Homebrew and some npm packages. If you see an error mentioning xcode-select, run this and wait for it to finish before retrying.

Terminal — zsh
$ xcode-select --install
xcode-select: note: install requested for command line developer tools
# a dialog will pop up -- click Install and wait ~5 minutes
$ xcode-select -p # verify it worked
/Library/Developer/CommandLineTools ✓
06
Claude installs but nothing happens when you run it

If claude runs but immediately exits or hangs, the install may be corrupted. A clean reinstall fixes this in most cases.

Terminal — zsh
$ which claude # find the install location
/usr/local/bin/claude
$ npm uninstall -g @anthropic-ai/claude-code
$ curl -fsSL https://claude.ai/install.sh | bash
$ claude doctor
✓ All checks passed
💡
Your config in ~/.claude/ is preserved during reinstall, so you won't lose any settings or authentication.
07
Still stuck? Run the full diagnostic

If none of the above resolves it, run the diagnostic script below and share the output in #tools-and-infra. It checks every known config point and tells you exactly what's wrong.

Terminal — zsh
$ echo "=== Shell ===" && echo $SHELL
$ echo "=== Node ===" && node --version
$ echo "=== npm ===" && npm --version
$ echo "=== Claude path ===" && which claude
$ echo "=== Claude version ===" && claude --version
$ claude doctor
# copy everything above and paste it in #tools-and-infra
